# Portfolio Value-at-Risk Backtesting Engine

A reproducible Python risk engine for estimating and backtesting daily portfolio Value-at-Risk (VaR) and Expected Shortfall (ES). The project uses a simple 50/50 AAPL-MSFT portfolio to demonstrate tail-risk estimation, rolling model validation, statistical coverage tests, and clean Python project structure.

## Why VaR Matters

Value-at-Risk estimates the return threshold that should only be breached with a chosen probability over a specified horizon. A 99% one-day VaR of -4.5%, for example, means daily losses worse than 4.5% are expected about 1% of the time under the model.

VaR is widely used in market risk because it is simple to communicate and backtest. It does not estimate how severe losses are once the threshold is breached, so this project also reports Expected Shortfall, which measures the average return in the tail beyond VaR.

## Methods Implemented

- Historical VaR
- Parametric Gaussian VaR
- Monte Carlo VaR calibrated to empirical mean and volatility
- Historical, Gaussian, and Monte Carlo Expected Shortfall
- Rolling 252-trading-day VaR backtesting
- Kupiec unconditional coverage test
- Christoffersen independence test
- Basel-style traffic light classification

## Dataset And Portfolio

- Assets: AAPL and MSFT
- Source: Yahoo Finance via `yfinance`
- Date range: 2018-01-01 to 2023-12-31
- Prices: adjusted close where available
- Portfolio: 50/50 daily rebalanced return series
- VaR horizon: one trading day
- Rolling estimation window: 252 trading days

## Interpretation

The 95% Historical model is slightly conservative by breach count, with 57 breaches versus 62.8 expected. Its Kupiec p-value does not indicate a coverage failure, and the Christoffersen p-value does not show strong evidence of breach clustering.

The 95% Parametric Gaussian and Monte Carlo models match the expected number of breaches closely, but the Christoffersen p-value is below 5%, suggesting breaches may not be independent through time. This is a common weakness of constant-volatility Gaussian models during clustered volatility regimes.

At 99%, the Historical model has 19 breaches versus 12.56 expected. That is higher than ideal, but the Kupiec p-value is 0.0896 and the simple scaled traffic-light rule remains green.

The 99% Parametric Gaussian and Monte Carlo models are weaker for this portfolio and sample. Both produce 31 breaches versus 12.56 expected, with a very low Kupiec p-value and yellow traffic-light classification. This suggests the Gaussian tail assumption underestimates extreme downside moves.

## Limitations

- The portfolio contains only two large-cap technology stocks.
- Gaussian parametric VaR can underestimate fat-tailed losses.
- Historical returns may not predict future crisis behavior.
- The model does not include transaction costs because this is a risk model, not a trading strategy.
- Monte Carlo returns are simulated from a normal distribution calibrated to the rolling training window, so this implementation is intentionally simple rather than a full market simulator.
